# QS=CD ルール仕様

本書では、カードゲーム「QS=CD」のルールを定義する。

## 1. 思想・前提

- QS = CD
- Q = 1
- したがって、S = CD
- C ≥ 1
- D ≥ 1
- Q（品質）は固定値1であり、プレイヤーが操作できる数値としては扱わない。
- 品質は交渉の対象ではなく、常に満たすべき前提条件とする。
- プレイヤーが管理する対象は、S（スコープ）である。
- C（コスト）とD（デリバリー）は、Sを実現するために必要な制約である。
- 1のSを実現するには、それに見合うCとDが必要となる。
- CやDを十分に確保せずにSを追求する行為は、本来成立しない要求を押し付けることを意味する。
- その結果として発生する負荷や疲弊を、本ゲームの思想上ではQの低下として捉える。ただし、これは思想上の表現であり、ゲーム上のQが数値として変動することを意味しない。
- 成果物の品質が維持されていても、関係者のQOLが損なわれている状態は望ましくない。

## 2. ゲーム概要

- 基本プレイ人数：1人
- プレイヤーの役割：PM（プロジェクトマネージャー）
- ゲーム開始時に、プロジェクトモード、メンバーカードセット、継続カードセットを選択する。
- 選択したプロジェクトモードとカードセットは、そのプロジェクトの終了まで変更できない。
- 目的：スコアとコストを両立し、次の条件を満たすこと
  - 最終スコアが目標スコア以上である。
  - 最終コストがコスト制約以下である。

## 3. 変数

### S（スコープ）

- 成果量を表す。
- ゲーム上ではスコアとして集計する。

### C（コスト）

- 場に存在するメンバーカードの枚数を表す。
- 手札カードはコストに含めない。
- 表向きになったメンバー離脱カードは、例外としてコストに含めない。
- 「サビ残強制」を行った追加ラウンドで配られたメンバーカードは、例外として最終コストに含めない。

### D（デリバリー）

- ラウンド数を表す。
- 基本ラウンド数は3ラウンド、最大ラウンド数は5ラウンドとする。

### Q（品質）

- 固定値1とする。
- プレイヤーが操作する数値としては扱わない。

## 4. 初期設定

標準ルールでは、次の値でゲームを開始する。

- チーム人数：5人
- 1人あたりのスコア見込み：4点
- 目標スコア：63点
- コスト制約：30
- プロジェクトモード：単発プロジェクト
- メンバーカードセット：安定型
- 継続カードセット：標準

各設定値はカスタマイズできる。チーム人数は3〜6人、1人あたりのスコア見込みは3〜6点の範囲で設定する。

1人あたりのスコア見込みは、すべてのメンバーに共通の値とする。ゲーム中は固定され、メンバーごとには変化しない。

## 5. ラウンド構造

- 1ラウンドを1行で表す。
- 横方向には、チーム人数分のメンバー列を配置する。
- 1列が1人のメンバーに対応する。
- 行は、下から上へ時間順に積み重ねる。
- 各行の列数は、ゲーム開始時に設定したチーム人数分で固定する。
- 離脱済みのメンバー列にはカードを配らず、空白として扱う。

## 6. メンバーカード

メンバーカードは、安定型とハイリスク型の2種類のセットから選択する。どちらも数値カードと特殊カードを合わせた35枚で構成する。

### 6.1. 安定型

安定型は、標準ルールで使用するメンバーカードセットとする。

| カード | 枚数 |
| ---: | ---: |
| 0 | 2枚 |
| 1 | 3枚 |
| 2 | 4枚 |
| 3 | 6枚 |
| 4 | 10枚 |
| 5 | 7枚 |
| 6 | 2枚 |
| メンバー離脱 | 1枚 |
| 合計 | 35枚 |

### 6.2. ハイリスク型

ハイリスク型は、低スコア、高スコア、メンバー離脱の発生率を安定型より高めたメンバーカードセットとする。高スコアカードである5〜6の合計枚数は、低スコアカードである0〜2の合計枚数より多くする。

| カード | 枚数 |
| ---: | ---: |
| 0 | 3枚 |
| 1 | 3枚 |
| 2 | 4枚 |
| 3 | 4枚 |
| 4 | 6枚 |
| 5 | 8枚 |
| 6 | 4枚 |
| メンバー離脱 | 3枚 |
| 合計 | 35枚 |

## 7. メンバーカードの配布

各ラウンドの開始時に、次の手順でメンバーカードを配る。

1. チーム人数分のメンバー列を確認する。
2. 離脱していないメンバー列に、山札からメンバーカードを1枚ずつ配る。
3. 配ったカードをすべて裏向きにし、そのラウンドの行へ横一列に配置する。

離脱済みのメンバー列にはカードを配らず、空白として扱う。

## 8. メンバーカードを表にする

### 8.1. 表にする枚数

- 各ラウンドでは、規定された枚数のメンバーカードを必ず表にする。
- 表にするカードは、そのラウンドに裏向きで配られたカードの中から任意に選ぶ。
- カードを表にしないという選択はできない。
- 実際に表にする枚数は、次のうち少ない方とする。
  - 算出された上限枚数
  - そのラウンドで実際に配られたメンバーカードの枚数

### 8.2. 表にする枚数の上限

1人あたりのスコア見込みに応じた基本上限は、次のとおりとする。

| 1人あたりのスコア見込み | 基本上限 |
| ---: | ---: |
| 3 | 1枚 |
| 4 | 2枚 |
| 5 | 3枚 |
| 6 | 4枚 |

次の条件に該当する場合は、基本上限に枚数を加算する。

- メンバーカードを表にする時点で、離脱していないメンバーが6人の場合：+1枚
- 追加ラウンドの場合：+1枚

表にする枚数が増えることは、人数の増加や無理な進行によって、プロジェクトの不確実性が高まっている状態を表す。

## 9. メンバーカードの処理

ラウンド終了時に、そのラウンドのメンバーカードを次のとおり処理する。

### 表向きのカード

- カードに記載された数値または効果を、スコアまたはカード効果として扱う。

### 裏向きのカード

- 初期設定で決めた、全メンバー共通のスコア見込みをスコアとして扱う。

### 離脱済みのメンバー列

- カードは存在しない。
- コストは0とする。
- スコアは0とする。

## 10. メンバー離脱

- メンバー離脱カードが表向きになった場合、次のラウンド以降、該当するメンバー列にはカードを配らない。
- 表向きになったメンバー離脱カードは、そのラウンドのコスト・スコアともに0として扱う。
- 離脱後の該当メンバー列は、次のラウンド以降、コスト0・スコア0とする。
- メンバー離脱カードは、手札の「無効化」によって、そのラウンドに限り無効化できる。
- 無効化したメンバー離脱カードは、そのラウンドでは裏向きのカードとして扱う。
- 無効化した場合、離脱効果は発生しない。

## 11. 手札

### 11.1. 基本ルール

- 各ゲームは、手札5枚で開始する。
- ゲーム中のラウンド間では、手札を補充しない。
- 継続ゲームの開始時には、手札を再び5枚に補充する。
- 1ラウンドに使用できる手札は、最大2枚とする。
- 使用した手札は、使用したラウンドの場に残す。
- 手札カードはメンバーカードではないため、コストに含めない。

各ラウンドの開始時、メンバーカードを表にする前に、次の内容を決定する。

- そのラウンドで使用する手札の枚数（0〜2枚）
- 各手札の使用方法

### 11.2. 手札の使用方法

#### 3点スコアとして使用する

- 手札1枚を3点スコアとして、そのラウンドの場札の右側に出す。
- このカードはコストに含めない。
- プレイングマネージャーとして、PMが直接成果を出す行為を表す。

#### 無効化として使用する

- 手札1枚を使用し、表向きにしたメンバーカード1枚の効果を無効化する。
- 無効化したカードは、そのラウンドでは裏向きのカードとして扱う。
- スコア計算には、初期設定で決めた全メンバー共通のスコア見込みを使用する。
- マネージャーとして、PMがトラブルを予防する行為を表す。
- 無効化として手札を使用すること自体は、メンバーカードを表にする前に宣言する。
- 無効化の対象は、メンバーカードを表にした後で選択する。
- 無効化を2枚分宣言した場合でも、対象にできるカードが1枚しかなければ、残りの1枚分は効果を発揮しない。
- 同じメンバーカードを、複数の無効化の対象にすることはできない。
- 数値カードを無効化した場合、その数値の代わりに、初期設定で決めた全メンバー共通のスコア見込みを使用する。

## 12. 追加ラウンド

- 3ラウンド終了時に、保持している継続カードの補正を適用したスコアが目標スコアへ到達していない場合に限り、追加ラウンドを実施できる。
- 追加ラウンドは最大2ラウンドとし、ゲーム全体では最大5ラウンドとする。
- 4ラウンド終了時に、保持している継続カードの補正を適用したスコアが目標スコアへ到達した場合、その時点でゲームを終了し、5ラウンド目は実施しない。
- 4ラウンド終了時点でも目標スコアへ到達していない場合に限り、5ラウンド目を実施できる。
- 追加ラウンドでは、メンバーカードを表にする枚数の上限に+1枚する。
- 追加ラウンドへ移行するときには継続カードを引かない。継続カードを引くのは、すべてのラウンドを終えてゲームが終了したときのみとする。

### 12.1. サビ残強制

- 追加ラウンドで、本来発生するコストを支払わない行為を「サビ残強制」と定義する。
- サビ残強制を行った追加ラウンドでは、そのラウンドで新たに配られたメンバーカードを最終コストに含めない。
- サビ残強制を行っただけでは、即敗北にはならない。
- サビ残強制を行ったゲームの終了時に継続カード「監査」を引いた場合、そのゲームはスコアに関係なく即敗北となる。

## 13. ゲーム終了時の処理

ゲーム終了時は、次の順序で処理する。

1. 各ラウンドの結果から基本スコアを計算する。
2. すでに保持している継続カードの補正を適用する。
3. 最終スコアを確定する。
4. 勝利条件を判定する。
5. 継続カードを引き、その効果を処理する。

手順4で通常の勝利条件を満たした場合でも、そのゲームでサビ残強制を行っており、手順5で継続カード「監査」を引いた場合は、特殊処理として通常の勝利判定を覆し、敗北とする。

## 14. 勝利条件

次の条件を両方満たした場合、ゲームに勝利する。

- 最終スコアが目標スコア以上である。
- 最終コストがコスト制約以下である。

## 15. 継続ゲーム

- 継続ゲームを行うのは、大型プロジェクトとエンドレスプロジェクトのみとする。
- ゲーム終了時に、離脱していない各メンバーについて継続カードを1枚ずつ引く。
- 離脱済みのメンバーについては、継続カードを引かない。
- メンバーごとに引いた継続カードの「対象メンバー」は、そのカードを引いたメンバーとする。
- 継続カードの効果は累積する。
- 次のゲームの目標スコアは、直前のゲームの最終スコア以上に設定する。
- 継続ゲームの開始前に、PMはメンバーの入れ替えやチーム人数の変更を行うことができる。
- メンバーの入れ替え、追加、離任などによってチーム構成が変わった場合、チームスコアアップの効果は失効する。
- 最初のゲームで選択したメンバーカードセットと継続カードセットは、継続ゲーム開始時に変更できない。

## 16. 継続カード

継続カードは、標準とWの2種類のセットから選択する。Wは標準の各カードを2倍にした構成とする。

| カード | 標準 | W |
| --- | ---: | ---: |
| 監査 | 1枚 | 2枚 |
| 離任 | 1枚 | 2枚 |
| チームスコアアップ | 2枚 | 4枚 |
| メンバーコストアップ | 3枚 | 6枚 |
| メンバースコアアップ | 3枚 | 6枚 |
| コスト削減圧力 | 1枚 | 2枚 |
| 何もなし | 3枚 | 6枚 |
| 合計 | 14枚 | 28枚 |

### 16.1. 監査

- サビ残強制を行ったPMに対する事後的な罰を表す。
- そのゲームでサビ残強制を行っていた場合、スコアに関係なく即敗北となる。
- 場には残らない。

### 16.2. 離任

- 対象のメンバーは、次のゲーム以降のチームから外れる。
- 場には残らない。

### 16.3. チームスコアアップ

- 最終スコアにチーム人数分のスコアを加算する。
- メンバーの入れ替え、追加、離任などによってチーム構成が変わった場合、効果は失効する。
- 継続ゲームでも場に残る。

### 16.4. メンバーコストアップ

- 対象メンバーの最終コストに、カード1枚につき1を加算する。この加算は、そのメンバーへ配られた各メンバーカードに対して行うものではない。
- 継続ゲームでも場に残る。

### 16.5. メンバースコアアップ

- 対象メンバーの最終スコアに2点を加算する。
- 継続ゲームでも場に残る。

### 16.6. コスト削減圧力

- 上位組織からのコスト削減要求を表す。
- 次のゲーム以降、コスト制約を恒久的に1減らす。
- コスト制約の減少は累積する。
- 場には残らない。

### 16.7. 何もなし

- 効果はない。
- 場には残らない。

### 16.8. 継続カード山札の再構築

- 各ゲームの終了時に、場に残らない継続カードと、山札に残っている継続カードを合わせてシャッフルし、新しい継続カード山札とする。
- 離任やチーム構成の変更によって失効した継続カードも、場に残らない継続カードとして扱う。

## 17. メンバーカード山札の再構築

継続ゲームを開始するときは、直前のゲームで使用したすべてのメンバーカードを回収してシャッフルし、プロジェクト開始時に選択した35枚のメンバーカード山札へ戻す。

## 18. 継続カードの配置

### メンバー系カード

次のカードは、1行目の下にある、対象メンバーの列へ配置する。

- メンバーコストアップ
- メンバースコアアップ

### チーム系カード

次のカードは、チーム系カード専用の領域へ配置する。

- チームスコアアップ

## 19. プロジェクトモード

プロジェクト開始時に、単発プロジェクト、大型プロジェクト、エンドレスプロジェクトのいずれかを選択する。選択したプロジェクトモードは、そのプロジェクトの終了まで変更できない。

### 19.1. 単発プロジェクト

- 1ゲームで終了する。
- 通常の勝利条件を満たした場合、プロジェクトクリアとする。
- ゲーム終了時には通常どおり、離脱していない各メンバーについて継続カードを1枚ずつ引く。
- そのゲームでサビ残強制を行っており、継続カード「監査」を引いた場合、通常の勝利判定を覆してプロジェクト失敗とする。
- 監査以外の継続カードは、次のゲームが存在しないため効果を適用しない。

### 19.2. 大型プロジェクト

- 全3ゲームで構成する。
- 初期ゲームのコスト制約は30とする。
- 各ゲームには通常の勝利条件を適用する。
- いずれかのゲームでスコア不足、コスト超過、または監査敗北が発生した場合、その時点でプロジェクト破綻として終了する。
- 各ゲームの最終スコアと最終コストをそれぞれ累積する。
- 3ゲームを破綻せずに終了し、3ゲームの最終スコア合計が200点以上の場合、プロジェクトクリアとする。
- 3ゲームを破綻せずに終了しても、3ゲームの最終スコア合計が200点未満の場合、プロジェクト失敗とする。
- 最終効率評価は、次の式で求める。

```text
最終効率評価 = 3ゲームの最終スコア合計 ÷ 3ゲームの最終コスト合計
```

- コスト削減圧力を引いた場合、次のゲーム以降のコスト制約へ累積して適用する。
- 3ゲーム目の終了時にも継続カードを引く。監査は通常どおり処理し、監査以外の継続カードは次のゲームが存在しないため効果を適用しない。

### 19.3. エンドレスプロジェクト

- プロジェクトが破綻するまで継続ゲームを繰り返す。
- 各ゲームには通常の勝利条件を適用する。
- スコア不足、コスト超過、または監査敗北が発生した場合、プロジェクト破綻として終了する。
