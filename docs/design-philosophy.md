# QS=CD 設計思想

本書では、カードゲーム「QS=CD」の背景と、各ルールが表す意味を説明する。ゲーム処理の正本は[`rule.md`](rule.md)とし、本書はルールを追加または変更するものではない。

## 1. QS = CD

QS=CDは、次の前提に基づく。

- `QS = CD`
- `Q = 1`
- `S ≥ 1`
- `C ≥ 1`
- `D ≥ 1`

Q（品質）は固定値1であり、交渉、削減または調整の対象にしない。CとDを伴って成立する範囲を定めるという意味で、本質的な調整対象はS（スコープ）だけである。

C（コスト）とD（デリバリー）は、Sを外部から制限するだけの条件ではない。CとDはSを成立させるために不可欠な構成要素であり、1のSを成立させるには、1以上のCと1以上のDが必要となる。CまたはDを伴わないSは成立しない。

`QS = CD`は、ゲーム上のスコアを算出するための数式ではない。成果だけをコストやデリバリーから切り離して要求することはできない、という本作の設計思想を表す。

一方、本作でプレイヤーが担うPMには、プロジェクト条件として与えられたSをゲーム中に変更する権限がない。本来の調整対象と、PMに与えられた権限の間にある差も、本作が扱う主題の一つである。

## 2. 想定するプロジェクト

本作は、期間型のソフトウェア開発プロジェクトを題材とする。

ゲームは、目標となるSを含むプロジェクト条件が定まった後から始まる。

PMとメンバーは、プロジェクトのために雇用または配置され、与えられた目標、コストおよび期限のもとで仕事をする。PMは事業オーナーではなく、プロジェクトを実施するか、目標に事業価値があるか、十分な予算が与えられるかといった前提を自由に決定できない。

プレイヤーが操作できる範囲を限定しているのは、この立場を表すためである。PMは目標となるSや成果を直接決定するのではなく、限られた情報と手段を使い、不確実性へ対応する。

## 3. プロジェクトの終了

本作では、プロジェクトを次の2種類として捉える。

1. 破綻したプロジェクト
2. 破綻する前に必要な成果を満たして終了したプロジェクト

これは一般的なプロジェクトマネジメント理論の定義ではなく、QS=CDが採用する作品上の公理である。

プロジェクトクリアは、永続的な成功を証明するものではない。決められた期間に必要なSを成立させ、破綻する前にプロジェクトを終了できたことを意味する。プロジェクトを適切に終了させることも、PMが担うべき役割の一つである。

単発プロジェクトと大型プロジェクトは、終了地点まで成立を維持できるかを扱う。エンドレスプロジェクトは終了地点を設けず、プロジェクトが破綻するまでの継続を扱う。

## 4. 見込みと不確実性

裏向きのメンバーカードは、不確実性の影響が顕在化せず、予定どおりに共通のスコア見込みを達成した状態を表す。

表向きのメンバーカードは、不確実性の影響を受けた結果を表す。その結果は、見込みと同じ場合、見込みを上回る場合、見込みを下回る場合、メンバーが離脱する場合がある。予定どおりに進む可能性もあるが、多くの場合、不確実性は予定どおりの進行を妨げる要因となる。PMは結果そのものを決定できず、結果が明らかになった後の対応を選択する。

1人あたりのスコア見込みが高いほど、表にするカードの基本上限は増える。これは、高い要求水準によって不確実性が増すことを表す。チーム人数が6人の場合や追加ラウンドでも上限が増え、人数と期間の増加による不確実性を表す。

## 5. PMの手札

PMが全力でワーカーとして稼働した場合、1ラウンドに6点、基本3ラウンドで合計18点の成果を出せるものと考える。手札1枚を3点として使用するため、18点は本来6枚分に相当する。

この6枚分のうち、1枚分をプロジェクトの進捗観測へ割り当てる。進捗観測へ使用する1枚分はプレイヤーの手札には含めず、残る5枚をゲーム上の手札とする。進捗観測の1枚分は設計思想上の表現に限り、実装上のカードやゲーム状態としては扱わない。

進捗観測は、PMが継続的に行う管理活動を表す。メンバーカードを表にする処理は進捗観測そのものではなく、プロジェクトの不確実性が結果へ影響することを表す。

すべてが見込みどおりに進み、与えられた制約にも余裕があれば、PMは手札を使って介入する必要がない。本作では、目標、コストおよび期間の制約と不確実性によって、そのままでは必要なSを成立させられない状況を扱う。そのときPMは、限られた手札を成果またはリスク対応へ使う。

手札を使うこと自体がPMの成功を意味するわけではない。介入せずに必要なSが成立するなら、それが最も余力を残した状態である。

手札を3点スコアとして使用することは、PMがプレイングマネージャーとして直接成果を出す行為を表す。直接成果を出すほど、同じ手札をリスク対応には使用できなくなる。

手札を無効化として使用することは、PMが事前対応によってトラブルの発生を防ぐ行為を表す。無効化の宣言をカードの公開前に行うのは、問題が明らかになってから過去へ遡って予防することはできないためである。無効化は、問題の隠蔽や実績値の書き換えを意味しない。

## 6. 継続による変化

本作の主題は、継続ゲームによってより明確になる。

メンバーの離任、コスト増加、スコア増加、チームの成長およびコスト削減圧力は、ゲームをまたいで影響する。一度のゲームを成立させることと、プロジェクトを継続して成立させることは同じではない。

単発プロジェクトでも継続カードを引くが、次のゲームが存在しないため、監査以外の効果は適用されない。これは影響が存在しないことではなく、現在のプロジェクトとPMの評価期間外へ出たことを表す。

## 7. サビ残強制と監査

サビ残強制は、追加ラウンドの成果を得ながら、本来必要なコストをプロジェクト上のコストとして計上しない行為を表す。

短期的には目標達成へ近づくが、監査によって通常の勝利判定が覆る可能性がある。これは、コストを伴わないSが成立したことを意味しない。支払われるべきCをプロジェクトの外側へ押し出した状態である。

## 8. 標準設定とカスタマイズ

標準設定は、基本ルールを習得しやすくするため、コスト制約に余裕を持たせた基準値である。標準設定だけを唯一の難易度とはしない。

目標スコア、コスト制約、チーム人数、1人あたりのスコア見込み、カードセットおよびプロジェクトモードを組み合わせることで、異なる課題と難易度を設定できる。デジタル版では、デイリーミッションなどの定められた設定セットとして提示することも想定する。

これらの設定は、プレイヤーまたはシナリオがゲーム開始前にプロジェクト条件を定めるためのものであり、ゲーム内のPMが目標を自由に変更することを意味しない。プロジェクト開始後、PMは与えられたSを変更せず、その成立に向けて対応する。
